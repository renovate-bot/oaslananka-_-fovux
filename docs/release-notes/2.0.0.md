# Fovux 2.0.0 Release Notes

Fovux 2.0.0 turns the v1 workbench into a safer, more reliable local product for YOLO practitioners.
Publishing remains maintainer-gated: source installs and local VSIX packaging are the supported path
until PyPI, VS Code Marketplace, and Open VSX are explicitly approved.

## Headline Wins

- **Secure local HTTP transport:** every endpoint except `/health` now requires a bearer token in
  `FOVUX_HOME/auth.token`, tool calls are rate-limited, and the server can run UNIX-socket-first on
  Unix systems.
- **More truthful exports and training lifecycle:** ONNX parity no longer reports false success,
  training writes deterministic params plus `pid.txt`, and `metrics.jsonl` is the primary live
  metrics schema.
- **Daily-use Studio UX:** Fovux Studio includes live webviews, a training launcher, run stop/resume/
  delete/tag actions, comparison views, and export history visibility.

## Breaking Changes

- Start HTTP explicitly for Studio on TCP while the extension transport is finalized:

```bash
fovux-mcp serve --http --tcp --metrics
```

- HTTP clients must send the token:

```bash
curl -H "Authorization: Bearer $(cat ~/.fovux/auth.token)" http://127.0.0.1:7823/runs
```

- Rotate a leaked local token with:

```bash
fovux-mcp rotate-token
```

- Training metadata now stores runtime PID in `pid.txt`; do not depend on `params.json.pid`.

## Upgrade Path

```bash
cd fovux-mcp
uv sync --locked --extra dev
cd ../fovux-studio
pnpm install --frozen-lockfile
pnpm verify
pnpm dlx @vscode/vsce@3.9.1 package --out fovux-studio.vsix --no-dependencies
```

See `fovux-mcp/docs/upgrade/v2.md` for the full migration guide.
